# Financial Ledger Decision\n\n## Current source of balance\n\nFor the current backend phase, `Account.currentBalance` is the operational source of account balance.\nFinancial modules update this cached balance directly inside their service transactions.\n\n## Transaction ledger status\n\nThe Prisma schema contains a `Transaction` model intended to become an immutable ledger, but the backend does not feed it uniformly yet.\nBecause of that, reports and dashboard calculations must not assume `Transaction` is complete.\n\n## Modules that currently change balances directly\n\n- `Revenue`: increments account balance when revenue is received.\n- `Expense`: decrements account balance when expense is paid.\n- `Bill`: decrements account balance when bill is paid.\n- `Transfer`: decrements the origin account and increments the destination account.\n- `EmployeePayment`: decrements account balance when an account is linked to the payment.\n\n## Future phase\n\nA future ledger hardening phase should make `Transaction` the durable audit trail for all balance-changing operations, including corrections and reconciliation.\nUntil then, service-level balance tests are required to protect `Account.currentBalance` behavior.\n
